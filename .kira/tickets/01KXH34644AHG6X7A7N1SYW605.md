codec.emitQuoted emits YAML DoubleQuotedStyle: title 'he said "hi"' -> 'title: "he said \"hi\""'. internal/core/eventparse.go:119 unquoteScalar is strings.Trim(TrimSpace(v), `"'`) which yields 'he said \"hi\' — escapes kept, trailing chars eaten. Corruption occurs for values containing '"', '\', edge quote chars, or non-printables (plain quoted values round-trip). Values flow into SQLite events (index.replaceEvents) and out via kira log/blame; blame.go scalarFieldValues re-serializes via codec.Serialize then re-parses with the same broken helper — any codec quoting change silently diverges both.

Fix: move scalar-line decoding into codec (codec.DecodeScalar via yaml.Unmarshal into a string — the exact inverse of EmitScalar) and use it in eventparse.frontmatterField and blame; delete unquoteScalar. Two required special cases: (1) null — blame.go:65 relies on the literal string "null" (from writeReqNullableScalar's 'epic: null') surviving parsing, but yaml.Unmarshal("null", &string) yields "" and would silently break the null-skip; (2) a fallback for unparseable lines, since frontmatterField consumes diff hunks of possibly hand-edited YAML.

Verify: round-trip table test EmitScalar -> DecodeScalar for escapes/quotes/null; a blame test over a title containing a double quote.

Files: internal/core/eventparse.go, internal/core/blame.go, internal/codec/serialize.go
