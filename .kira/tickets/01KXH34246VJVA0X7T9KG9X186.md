internal/workon/workon.go:92 builds '(?i)\b<key>-(\d+)\b' but normalize() with CasingSnake emits 'kira_142' (proven by the package's own test, test/workon_test.go:36-38); '_' is a \w char so even the boundary fails. branchPrefix (:58-67) only truncates at {slug}: for '{key}/{number}' the full 'kira/kira-142' becomes a prefix and HasPrefix accepts 'kira/kira-1420'; a slug-first pattern yields prefix "" so MatchBranch returns branches[0] (e.g. main) and core/workon.go checks it out (found=true from MatchBranch at core/workon.go:33, checkout via checkoutBranch at :49 / ensureWorktree at :41). Config validation only requires {number}. Test matrix covers only kebab + '-{slug}' patterns — exactly the untested cells. ParseActive's permissive fallback (:110-121, legacy return :119-120) turns corrupt/ticketless JSON into ok=true with the raw blob as Ticket. Package imports all of datamodel for a two-value casing enum.

Fix: build an anchored regex from the full rendered pattern (QuoteMeta parts, {slug} -> .*, [-_] separator, explicit boundary classes) shared by MatchBranch and InferNumber; slug-less patterns degenerate to exact match; empty prefix means no-match. ParseActive returns ok=false for JSON with empty Ticket; gate the legacy branch on a bare-token shape. Optionally take a separator string instead of datamodel.Casing.

Verify: convert tests to a casing x pattern table (snake, {key}/{number}, {slug}-{number}, RenderWorktreeDir, corrupt JSON).
